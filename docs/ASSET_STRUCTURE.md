# Asset Structure & Organization

This guide explains how assets (images, fonts, favicons) are organized in the micro-frontend demo project.

## ❌ Common Misconception

**NO - You do NOT copy the same asset to both places!**

You choose ONE location based on HOW you use the asset:

- **Importing in code?** → Use `src/assets/`
- **Referencing by URL?** → Use `public/assets/`

## 📁 Asset Folder Structure

### ✅ Correct Structure

**All Applications (Host & Remotes):**

```
app/
├── src/
│   └── assets/          # ✅ Assets IMPORTED in code (webpack processed)
│       ├── images/      #   - Processed and hashed by webpack
│       └── fonts/       #   - Output to dist/assets/fonts/[name].[hash:8][ext]
│                        #   Note: Favicons should go in public/assets/favicons/
│
├── public/
│   ├── index.html       # HTML template
│   └── assets/          # ✅ Static assets REFERENCED by URL (copied as-is)
│       ├── images/      #   - Copied as-is to dist/
│       ├── fonts/       #   - Reference in HTML: <link href="/assets/fonts/...">
│       └── favicons/    #   ✅ FAVICONS GO HERE (referenced in HTML)
│
└── dist/                # ✅ BUILD OUTPUT (generated by webpack)
    ├── index.html       # Compiled HTML
    ├── main.bundle.js   # JavaScript bundles
    ├── css/             # Extracted CSS
    └── assets/          # All assets end up here (from both src/ and public/)
        ├── images/      # From src/assets (hashed) + public/assets (original names)
        ├── fonts/
        └── favicons/    # Only from public/assets/favicons/ (original names)
```

## 🎯 When to Use Which Folder

### Use `src/assets/` for:

- ✅ **Images imported in code**: `import logo from '../assets/images/logo.png'`
- ✅ **Fonts loaded via CSS**: `@font-face { src: url('../assets/fonts/...') }`
- ✅ **Any asset imported in JavaScript/TypeScript files**
- ✅ **Benefits**: Webpack processes these, adds content hashes, optimizes them
- ❌ **NOT for favicons**: Favicons should always go in `public/assets/favicons/`

**Example:**

```tsx
// src/components/Header.tsx
import logo from "../assets/images/logo.png"; // ✅ From src/assets/

function Header() {
  return <img src={logo} alt="Logo" />;
}
```

### Use `public/assets/` for:

- ✅ **Favicons referenced in HTML**: `<link rel="icon" href="/assets/favicons/favicon.ico">`
- ✅ **Images referenced by URL in HTML**: `<img src="/assets/images/banner.jpg">`
- ✅ **Static assets that need to keep original filenames**
- ✅ **Benefits**: Copied as-is to output, maintain original paths

**Example:**

```html
<!-- public/index.html -->
<link rel="icon" href="/assets/favicons/favicon.ico" />
<!-- ✅ From public/assets/ -->
```

## ⚠️ Important: Favicons Location

**Favicons should ALWAYS go in `public/assets/favicons/`**, NOT in `src/assets/favicons/`.

**Why?**
- Favicons are referenced in HTML via `<link>` tags
- They need predictable URLs (not hashed filenames)
- Browsers and SEO tools expect them at specific paths
- They're not imported in JavaScript/TypeScript code

**✅ Correct:**
```
public/
└── assets/
    └── favicons/
        ├── favicon.ico
        ├── favicon-32x32.png
        └── favicon-96x96.png
```

**❌ Wrong:**
```
src/
└── assets/
    └── favicons/  ← Don't create this folder
```

## 📝 Usage Examples

### Images

**In code (imported from `src/assets/`):**

```tsx
// src/components/Logo.tsx
import logo from "../assets/images/logo.png"; // ✅ src/assets/

<img src={logo} alt="Logo" />;
// Webpack processes this and outputs: assets/images/logo.[hash:8].png
// In dist/: assets/images/logo.abc12345.png
```

**In HTML (referenced from `public/assets/`):**

```html
<!-- public/index.html -->
<img src="/assets/images/banner.jpg" alt="Banner" />
<!-- File must be in: public/assets/images/banner.jpg -->
<!-- Copied as-is to: dist/assets/images/banner.jpg -->
```

### Fonts

**In CSS (imported from `src/assets/`):**

```css
/* src/styles/fonts.css */
@font-face {
  font-family: "CustomFont";
  src: url("../assets/fonts/custom-font.woff2") format("woff2"); /* ✅ src/assets/ */
}
/* Webpack processes this and outputs: assets/fonts/custom-font.[hash:8].woff2 */
```

**In HTML (referenced from `public/assets/`):**

```html
<!-- public/index.html -->
<link rel="preload" href="/assets/fonts/static-font.woff2" as="font" />
<!-- File must be in: public/assets/fonts/static-font.woff2 -->
<!-- Copied as-is to: dist/assets/fonts/static-font.woff2 -->
```

### Favicons

**Favicons should ALWAYS go in `public/assets/favicons/`:**

Favicons are referenced in HTML, not imported in code. They need predictable URLs and should not be hashed.

```html
<!-- public/index.html -->
<!-- Place favicon files in public/assets/favicons/ -->
<link rel="icon" href="/assets/favicons/favicon.ico" />
<link
  rel="icon"
  type="image/png"
  sizes="32x32"
  href="/assets/favicons/favicon-32x32.png"
 />
<link
  rel="icon"
  type="image/png"
  sizes="96x96"
  href="/assets/favicons/favicon-96x96.png"
 />
<!-- Files must be in: public/assets/favicons/ -->
<!-- Copied as-is to: dist/assets/favicons/ -->
```

**❌ DON'T do this:**
```tsx
// Don't put favicons in src/assets/favicons/
import favicon from "../assets/favicons/icon.png"; // ❌ Wrong location
```

## ⚙️ Webpack Configuration

### Assets from `src/assets/` (Imported in Code)

Webpack automatically processes these when imported:

- **Images** (< 8kb): Inlined as base64
- **Images** (>= 8kb): Output to `dist/assets/images/[name].[contenthash:8][ext]`
- **Fonts**: Output to `dist/assets/fonts/[name].[contenthash:8][ext]`

**Benefits:**

- Content hashing (cache busting)
- Optimization
- Tree shaking (unused assets not included)

**Example:**

```tsx
import logo from "../assets/images/logo.png";
// logo will be: "assets/images/logo.abc12345.png" (with hash)
// In production: dist/assets/images/logo.abc12345.png
```

### Assets from `public/assets/` (Referenced by URL)

Webpack copies these as-is to the output:

- **All files**: Copied to `dist/assets/` maintaining folder structure
- **No processing**: Original filenames preserved
- **No hashing**: Exact path as in source

**Benefits:**

- Predictable URLs (good for favicons)
- No processing overhead
- Direct path references

**Example:**

```html
<img src="/assets/images/banner.jpg" />
<!-- File: public/assets/images/banner.jpg -->
<!-- Output: dist/assets/images/banner.jpg (same name) -->
```

## 📋 Decision Tree: Where Should I Put My Asset?

```
Do you import it in JavaScript/TypeScript code?
├── YES → Is it a favicon?
│   ├── YES → Use public/assets/favicons/ (favicons always go here)
│   └── NO → Use src/assets/
│       └── Example: import logo from '../assets/images/logo.png'
│
└── NO → Is it referenced by URL in HTML?
    ├── YES → Use public/assets/
    │   └── Example: <link rel="icon" href="/assets/favicons/favicon.ico">
    │
    └── NO → You probably don't need it, or it should go in src/assets/
```

## 📚 Summary Table

| Location         | Use For                  | Processing                | Build Output                          | Example                                                 |
| ---------------- | ------------------------ | ------------------------- | ------------------------------------- | ------------------------------------------------------- |
| `src/assets/`    | Assets imported in code  | Webpack processed, hashed | `dist/assets/[name].[hash:8][ext]`    | `import logo from "../assets/images/logo.png"`          |
| `public/assets/` | Assets referenced by URL | Copied as-is              | `dist/assets/[name][ext]` (same name) | `<link rel="icon" href="/assets/favicons/favicon.ico">` |

## 🎯 Best Practices

### ✅ DO

1. **Use `src/assets/`** for assets imported in JavaScript/TypeScript:

   ```tsx
   import logo from "../assets/images/logo.png";
   ```

2. **Use `public/assets/`** for assets referenced in HTML:

   ```html
   <link rel="icon" href="/assets/favicons/favicon.ico" />
   ```

3. **ALWAYS put favicons in `public/assets/favicons/`**:
   - Favicons are referenced in HTML, not imported in code
   - They need predictable URLs (not hashed)
   - Don't create `src/assets/favicons/` folder

4. **Keep folder structure consistent** across all apps

5. **Use descriptive filenames** for better organization

### ❌ DON'T

1. **Don't copy the same asset to both locations** - Choose one based on usage
2. **Don't mix approaches** - If you import an asset, put it in `src/assets/`
3. **Don't reference `src/assets/` by URL** - Use `public/assets/` for URL references
4. **Don't import assets from `public/assets/`** - Use `src/assets/` for imports
5. **Don't put favicons in `src/assets/favicons/`** - Always use `public/assets/favicons/`

## 🔄 Migration Guide

If you have assets in the wrong location:

### Moving from `public/assets/` to `src/assets/`:
1. Move file: `public/assets/images/logo.png` → `src/assets/images/logo.png`
2. Change import: `src="/assets/images/logo.png"` → `import logo from "../assets/images/logo.png"`
3. Update usage: `<img src={logo} />`

### Moving from `src/assets/` to `public/assets/`:
1. Move file: `src/assets/images/banner.jpg` → `public/assets/images/banner.jpg`
2. Update HTML: `import banner from ...` → `<img src="/assets/images/banner.jpg" />`

### For Favicons:
- ✅ Always use: `public/assets/favicons/`
- ❌ Never use: `src/assets/favicons/`

---

**Rule of thumb**: 
- If you're importing it in code → `src/assets/` (except favicons)
- If you're referencing it by URL → `public/assets/`
- **Favicons always go in `public/assets/favicons/`** - don't create `src/assets/favicons/`
